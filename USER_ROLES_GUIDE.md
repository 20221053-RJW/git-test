# 사용자 역할 시스템 가이드

## 개요

CampusConnect는 **학생(Student)**과 **교수(Professor)** 두 가지 역할을 지원합니다. 각 역할에 따라 보이는 기능과 권한이 다릅니다.

## 역할 유형

### 1. 학생 (Student)
- **기본 역할**: 대부분의 사용자
- **주요 기능**:
  - 팀 프로젝트 참여
  - 파일 업로드 및 공유
  - 팀원과 채팅
  - 프로젝트 진행률 확인
  - 자신의 프로필 관리

### 2. 교수 (Professor)
- **관리자 역할**: 과목 담당 교수
- **추가 기능**:
  - 팀원 평가 (학생 성적 입력)
  - 모든 팀의 프로젝트 확인
  - 학생 피드백 제공
  - 주차별 진행률 상세 분석

## 역할별 화면 차이

### 팀 상세 페이지 (`/app/teams/:id`)

#### 공통 기능
- 파일 트리 보기
- 프로젝트 실행 미리보기
- 주차별 진행률 차트
- 팀원 목록

#### 학생 전용
- 팀원 평가 불가
- "팀원 평가는 교수님만 가능합니다" 안내 메시지 표시

#### 교수 전용
- **팀원 평가** 버튼 활성화
- 개별 학생 평가 입력
  - 평균 기여도
  - 작업 완성도
  - 문제 해결력
  - 총체적 평가

## 개발 모드

### 역할 전환 기능
네비게이션 바 우측에 역할 전환 버튼이 표시됩니다:
- **학생** 버튼: 학생 모드로 전환
- **교수** 버튼: 교수 모드로 전환

> 이 기능은 개발 및 테스트 목적으로만 사용됩니다. 프로덕션에서는 로그인 시 역할이 자동 설정됩니다.

## 구현 상세

### AuthContext
```tsx
// src/app/contexts/AuthContext.tsx
const { 
  user,           // 현재 사용자 정보
  role,           // 현재 역할
  isStudent,      // 학생 여부
  isProfessor,    // 교수 여부
  setUserRole,    // 역할 전환 (개발용)
} = useAuth();
```

### 조건부 렌더링 예시

```tsx
// 교수만 표시
{isProfessor && (
  <button>팀원 평가</button>
)}

// 학생만 표시
{isStudent && (
  <p>평가는 교수님만 가능합니다</p>
)}
```

## 향후 확장

### 추가 예정 기능

1. **학생 뷰**
   - 자신의 평가 결과 확인
   - 팀원 상호 평가 (Peer Review)
   - 개인 기여도 통계

2. **교수 뷰**
   - 전체 과목 통계 대시보드
   - 학생별 성적 일괄 입력
   - 팀별 비교 분석
   - 평가 결과 엑셀 다운로드

3. **관리자 뷰** (향후)
   - 과목 생성 및 관리
   - 교수 권한 부여
   - 시스템 설정

## API 통합 시 고려사항

### 로그인 시 역할 설정
```tsx
// 로그인 응답 예시
{
  "user": {
    "id": "123",
    "name": "김교수",
    "email": "prof@example.com",
    "role": "professor"  // ← 서버에서 역할 전달
  }
}
```

### 권한 검증
- **프론트엔드**: UI 표시/숨김
- **백엔드**: API 레벨에서 권한 검증 필수

```tsx
// 예: 팀원 평가 API
POST /api/teams/:id/evaluate
Authorization: Bearer token

// 서버에서 role === 'professor' 확인
```

## 테스트 시나리오

### 학생 모드 테스트
1. 역할을 "학생"으로 전환
2. 팀 상세 페이지 접속
3. "팀원 평가" 버튼이 **표시되지 않음** 확인
4. 안내 메시지 표시 확인

### 교수 모드 테스트
1. 역할을 "교수"로 전환
2. 팀 상세 페이지 접속
3. "팀원 평가" 버튼 **표시됨** 확인
4. 평가 모달 정상 작동 확인

## 문제 해결

### 역할이 변경되지 않을 때
- 페이지 새로고침
- 브라우저 콘솔에서 `localStorage` 확인

### 권한 오류
```tsx
// useAuth는 AuthProvider 내부에서만 사용 가능
// App.tsx에서 AuthProvider로 감싸져 있는지 확인
<AuthProvider>
  <RouterProvider router={router} />
</AuthProvider>
```

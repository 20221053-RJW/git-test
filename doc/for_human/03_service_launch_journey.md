# 서비스 런칭까지의 여정

> **관련:** `doc/for_agent/04_service_launch_flow.md` · `doc/for_human/28_human_action_items.md`

“만들었다”와 “세상에 공개했다”는 다릅니다. 이 문서는 **공개 전에 거쳐야 할 관문**을 설명합니다.

## 비유: 식당 오픈

1. **메뉴 완성** (기능) — 주문·결제가 되는가  
2. **위생 검사** (보안) — 개인정보, 비밀번호  
3. **간판·주소** (배포) — 인터넷 주소  
4. **오픈 이벤트** (파일럿) — 한 수업만 먼저 써보기  
5. **정식 영업** (Launch) — 전체 공개  

## 체크리스트 (쉬운 버전)

### 제품
- [ ] 학생·교수가 실제로 쓸 핵심 3가지: 사람 찾기, 팀 작업, 내 기록
- [o] 수업·팀·Q&A 등 DB 읽기·쓰기 (facade)
- [o] 마이페이지 리포트 집계·A4 인쇄 (LLM 문단은 H-002)
- [ ] RLS 적용·프로덕션 배포 (H-001, H-005)

### 기술 (개발자 + AI)
- [ ] 비밀번호·API 키가 코드에 안 보이게
- [ ] “남의 수업 데이터를 못 보게” DB 규칙(RLS)

### 법·정책 (당신의 역할)
- [ ] 개인정보 처리방침
- [ ] 학교 규정에 맞는지 확인

### 공개 (당신의 승인)
- [ ] 스테이징(연습용 사이트)에서 테스트
- [ ] “이제 공개해도 된다”고 말해 주기

# 상태 관리 — “앱이 기억하는 것”

> **관련:** `doc/for_agent/08_frontend.md` · `doc/for_human/12_auth_system.md`

**상태(state)** = 지금 화면이 알고 있는 정보.  
예: 로그인 여부, 열린 모달, 팀 목록 배열.

## 종류

### 1. 전역 — AuthContext

- **누가 로그인했는지**  
- 모든 페이지에서 `useAuth()`로 접근  
- 파일: `contexts/AuthContext.tsx`  

### 2. 페이지 로컬 — useState

- 예: TeamsPage에서 “어떤 모달이 열렸는지”  
- **그 페이지를 벗어나면** 대부분 사라짐  

### 3. 서버 상태 (계획)

- DB에서 가져온 팀 목록  
- **React Query** 도입 시: 캐시·자동 새로고침  

## 비유

- **전역** = 출입증 (건물 어디서나 같은 사람)  
- **로컬** = 책상 위 메모 (그 방에서만)  
- **서버** = 창고 장부 (진실의 원천)  

## 흔한 혼란

“로그인했는데 다른 페이지에서 로그아웃된 것처럼 보임”  
→ AuthContext가 App 최상단을 감싸지 않았을 때 발생.  
현재는 `App.tsx`에서 Provider 사용 확인 필요.

## 앞으로

팀·수업 데이터도 전역이 아니라 **서버에서 가져와 캐시**하는 방식이 유지보수에 좋습니다.

기술: `doc/for_agent/08_frontend.md`
